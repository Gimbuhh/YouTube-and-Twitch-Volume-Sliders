Patch Notes - Version 2.0

### Compact In-Bar Volume Control

- Replaced the old floating resting overlay with a compact 40px circular in-bar control.
- The collapsed control shows the current volume with a YouTube-red SVG arc that starts at 12 o'clock.
- The center icon uses YouTube-style speaker states: muted/0% shows an x, 1-50% shows one bow, and 51-100% shows two bows.
- Clicking the speaker/arc icon toggles mute in both collapsed and expanded states.
- Hovering or focusing expands the same control to the right into a wide pill slider; collapse/expand uses a simple width-only animation so the icon stays fixed.
- The collapsed and expanded states share the same translucent pill background, avoiding flicker, double-rendered shadows, and zoom-sensitive backing shifts.
- The red volume arc uses the same smooth SVG circle/pathLength approach as the Twitch 2.0 slider, avoiding the aliasing from the temporary generated-arc experiments.
- The active pill gives the volume bar more horizontal travel for finer adjustments.
- The value label now uses a compact fixed, center-aligned slot, so the slider bar never changes size when switching between `5%`, `10%`, `100%`, or `Muted`.
- Tightened the value-label spacing and aligned the fill/tick visuals to the thumb-center travel span, so percentage positions stay visually consistent from 0% to 100%.
- Removed the old time-based snap workaround during drag; free dragging is the default again.
- Direct track clicks still snap to the nearest 5% point even when free dragging is selected.
- Added a `Slider behavior` section in the Volume Slider submenu with `Free drag` and `Snap to 5%` choices.
- Added an `Always expanded` setting for users who want the full pill visible without hovering the icon first.
- Added a `Show on video` setting that moves the 2.0 pill out of the YouTube controls bar and back onto the video surface.
- Restored `USER_SETTINGS.overlayOpacityUnfocused` and `USER_SETTINGS.overlayOpacityFocused` for the on-video placement only.
- After changing volume, the pill stays expanded until YouTube's controls hide.
- The 5% step marks stay visible whenever the pill is expanded, including the keep-open state after a volume change.
- Tuned the YouTube pill background to a more neutral dark translucent color so it better matches native YouTube control buttons.

### YouTube Control Bar Integration

- The slider now mounts directly in YouTube's `.ytp-left-controls` control bar.
- In normal mode, it appears after the time/chapter area as the last left-side item.
- In `Replace native` mode, it can mount either in the native volume-button position or in the normal custom-slider position.
- With `Show on video`, it mounts on the player surface like the older 1.5 overlay: low on the video when controls are hidden, and above the progress bar when controls are visible.
- The native YouTube volume area is hidden only when replacement mode is active.
- The control no longer uses small/big slider-size settings.
- Removed the standalone toggle button from the page controls.

### YouTube Settings Menu

- Added a `Volume Slider` row to YouTube's player settings menu.
- The row opens a compact submenu with `On`, `Off`, `Replace native`, and slider-behavior choices.
- Replacement mode includes `Native spot` and `Custom spot` placement choices.
- The settings are saved to browser storage:
  - `tm-yt-volume-slider-mode`
  - `tm-yt-volume-slider-location`
  - `tm-yt-volume-slider-replace-placement`
  - `tm-yt-volume-slider-snap-to-5`
  - `tm-yt-volume-slider-always-expanded`
- The settings can still be overridden in code through:
  - `USER_SETTINGS.volumeSliderMode`
  - `USER_SETTINGS.sliderLocation`
  - `USER_SETTINGS.replaceNativePlacement`
  - `USER_SETTINGS.snapToFive`
  - `USER_SETTINGS.alwaysExpanded`
  - `USER_SETTINGS.overlayOpacityUnfocused`
  - `USER_SETTINGS.overlayOpacityFocused`
- The submenu uses YouTube's native row/checkmark styling and updates in place without backing out to the root settings menu.
- The settings row is inserted below YouTube's toggle items and above the first submenu row (such as Sleep timer or Subtitles), and the menu height is fit to content to avoid scrollbars.
- Settings menu detection uses YouTube's menu structure instead of English label text, so the Volume Slider row appears in any UI language (for example Swedish or German).
- Menu width is measured from the current row content each time settings open, so the popup grows to fit long labels and values instead of clipping playback speed, quality, or other settings on the right.
- Fixed the `Always expanded` and `Show on video` rows so their active checkmarks and spacing match the other Volume Slider submenu options.
- The custom submenu now backs out to the main settings menu when YouTube controls hide, matching native YouTube submenu behavior.
- The Volume Slider submenu title now also acts as a back target, while the empty header area no longer shows a pointer cursor.

### Cleanup

- Removed obsolete slider-size USER_SETTINGS.
- Removed the standalone toggle-button path.
- Removed trial animation classes, SVG backing experiments, generated arc helpers, and redundant collapsed-state CSS.
- Kept the final implementation focused on one shared pill background, one SVG icon/arc stack, and one width-only expand/collapse path.
- Consolidated duplicate slider snap logic, pill expand/collapse styles, and submenu back-button handlers.
- Moved repeated volume-label styling from JavaScript into CSS.
- Removed redundant navigation-time settings injection and a duplicate expanded-state call in the slider change handler.
